Funcations

Welcome

  • Welcome

    1.数据存储：SharedPreference
 ⅰ.记录了登录状态
  LOGIN_STATUS
    value:0，1，2，3
  其中，各个数字代表的含义：
  0 -> 未曾登录过，经过初始化界面，直接进Welcome界面；
  1 -> 打开过Welcome界面，未登录，则经过初始化界面，直接进登录界面；
  2 -> 登录过，但是已经注销，只保留了帐号，消除了密码，经过初始化界面，直接进登录界面；
  3 -> 一直是登录状态，则经过初始化界面，跳过Welcome和登录界面，直接进到主界面。

OkHttpUtils

  • OkHttpUtils是 OkHttp的辅助类
  • 使用方法：使用GitHub上 hongyangAndroid大神的okhttputils的 教程。
  • 主要使用了GET和POST方法：

GET

 //以登录响应为例进行说明
@Override
    public void LoadLoginList(OnLoadModelListListener loadModelListListener) {
        OkHttpUtils
                .get()
                // Config.METHOD_URL:http://192.168.72.63:6686/Service1.svc/LoginCheck/tel/password
                // tel和password都是传出的参数，切记password后面没有斜杠
                .url(Config.METHOD_URL)
                .build()
                // 执行请求后，有个回调函数，返回字符串类型
                .execute(new StringCallback() {
            @Override
            public void onError(Call call, Exception e, int id) {
                // 响应失败的处理
                loadModelListListener.onFailuer(e);
            }
            @Override
            public void onResponse(String response, int id) {
                // 响应成功的处理
                List<LoginBean> loginBeen = JsonUtils.readJsonLoginBean(response);
                loadModelListListener.onSuccess(loginBeen);
            }
        });
    }

POST

 //以注册响应为例进行说明
@Override
    public void LoadRegList(String content, RegModel.OnLoadModelListListener loadModelListListener) {
        OkHttpUtils
                .postString()
                // Config.METHOD_URL:http://192.168.72.63:6686/Service1.svc/RegUser
                // 切记RegUser后面没有斜杠
                .url(Config.METHOD_URL)
                // 设置传出协议内容、格式
                .mediaType(MediaType.parse("application/json; charset=utf-8"))
                // 设置传出协议头内容、格式
                .addHeader("Content-type","application/json")
                .addHeader("Accept","application/json")
                .content(content)
                .build()
                // 执行请求后，有个回调函数，返回字符串类型
                .execute(new StringCallback() {
            @Override
            public void onError(Call call, Exception e, int id) {
                // 响应失败的处理
                loadModelListListener.onFailuer(e);
            }
            @Override
            public void onResponse(String response, int id) {
                // 响应成功的处理
                List<RegBean> regBeen = JsonUtils.readJsonRegBean(response);
                loadModelListListener.onSuccess(regBeen);
            }
        });
    }

FMMap

  • 使用 FMMap作为底图数据
    FMMap是北京蜂鸟视图科技有信公司制作的产品，专注于提供室内三维、二维地图、全景地图、商户POI兴趣点四类产品服务，使用户可以通过PC、只能手机等互联网终端对室内场景进行浏览。
  • SDK包括 Android SDK， iOS SDK， JavaScript SDK
    这里仅使用Android部分。蜂鸟地图 Android SDK（点 这里下载Android SDK）是一套基于Android 2.3及以上版本设备的应用程序接口。您可以使用该SDK开发适用于Android系统移动设备的室内地图应用，通过调用地图SDK接口，您可以轻松访问蜂鸟的地图服务和 数据，构建功能丰富、交互性强的室内地图类应用程序（这里有 代码示例）。
